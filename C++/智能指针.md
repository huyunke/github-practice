**智能指针**本质上是把指针封装进了一个对象里。当这个对象生命周期结束（比如函数运行完了）时，它的析构函数会自动帮我们调用 `delete`。这就是所谓的 **RAII（资源获取即初始化）**
- 智能指针本身不支持 `static_cast`, 但可以用 `.get()` 获取**裸指针**再进行 `static_cast`
#### std::unique_ptr
保证同一时间只有一个指针可以拥有这块内存
```cpp
#include <memory>

void uniquePtrDemo() {
    // 1. 创建：使用 std::make_unique 是最安全、最高效的方式
    std::unique_ptr<Trophy> myTrophy = std::make_unique<Trophy>("冠军金杯");

    // 2. 使用：像普通指针一样用 -> 或 *
    myTrophy->show();

    // 3. ⚠️ 尝试拷贝：这行会报错！编译器不允许两个 unique_ptr 指向同一个地方
    // std::unique_ptr<Trophy> copy = myTrophy; 

    // 4. 移动：如果你想转移所有权，必须用 std::move
    std::unique_ptr<Trophy> newOwner = std::move(myTrophy);
    
    // 此时 myTrophy 变成了 nullptr，newOwner 接管了内存
} // 函数结束，newOwner 自动析构，内存自动释放！
```

### std::shared_ptr
- **引用计数（Reference Counting）**：每当有一个新的 `shared_ptr` 指向同一个对象时，计数器 $+1$；当一个指针销毁或重指向别处时，计数器 $-1$。
    
- **自动销毁**：只有当计数器归 **0**（即没有任何人在看这本书）时，对象才会被真正 `delete`
```cpp
{
    auto sPtr1 = std::make_shared<GoldTrophy>("共享金杯"); // 计数 = 1
    {
        auto sPtr2 = sPtr1; // 计数 = 2
        std::cout << "当前有 " << sPtr1.use_count() << " 人拥有它\n";
    } // sPtr2 离开作用域，计数 = 1
} // sPtr1 离开作用域，计数 = 0 -> 自动释放内存
```

### std::weak_ptr(观察者)
- 它指向 `shared_ptr` 管理的对象，但**不增加引用计数**。它就像是远远地看着那本书，并不借走它

**为什么要它？** 因为“循环引用”会导致内存泄漏。 想象：**对象 A** 里有一个 `shared_ptr` 指向 **对象 B**，而 **对象 B** 里也有一个 `shared_ptr` 指向 **对象 A**

- 即使外部没有指针指向它们了，它们彼此还“抓着”对方，计数永远不会归 0
    
- **解决办法**：把其中一方改成 `weak_ptr`

- 因为 `weak_ptr` 不增加计数，它指向的对象**可能已经被销毁了**。 所以你不能直接用它，必须先用 `.lock()` 方法把它“升级”为 `shared_ptr`，如果升级成功，说明对象还在。
