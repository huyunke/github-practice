- 在传统的函数调用中，程序需要经历“跳过去执行函数 — 执行完再跳回来”的过程。这种“跳转”虽然很快，但如果函数体极其简单（比如只是返回一个数值），那么跳转所花费的时间可能比执行函数本身还要多

- `inline` 函数的核心思想是：**以空间换时间**

- 当你把一个函数声明为 `inline` 时，你是在向编译器发出一个“建议”：请把这个函数的代码直接“复印”到每一个调用它的地方，而不是通过跳转去调用

### `inline` 函数的特点

1. **减少开销**：省去了函数调用的参数压栈、跳转、返回等额外开销
    
2. **编译器决定权**：`inline` 只是一个**建议**。如果函数太复杂（比如包含循环、递归），编译器可能会忽略你的建议，依然把它当作普通函数
    
3. **定义必须可见**：编译器在编译调用点时需要知道函数的完整代码，所以 `inline` 函数通常定义在 **头文件** 中

### 代码实现
##### 1. 基本定义方式 

在函数返回类型前加上 `inline` 关键字即可。通常我们会将其放在头文件中

```cpp
// 在头文件 (.h) 中定义
inline int square(int x) {
    return x * x;
}
```

##### 2. 类内部的“隐式内联” 

在 C++ 中，如果你直接在类定义的内部编写成员函数的具体实现，编译器会自动将其视作 `inline` 函数，你甚至不需要手动写那个关键字


```cpp
class Player {
private:
    int score;
public:
    // 这种写在类内部的函数，默认就是 inline 的
    int getScore() const {
        return score;
    }
};
```

##### 3. 类外部的显式定义

如果你希望类定义保持整洁，也可以在类外部使用 `inline` 进行定义。

```
class Player {
    int score;
public:
    int getScore() const;
};

// 在头文件的下方进行内联定义
inline int Player::getScore() const {
    return score;
}
```
