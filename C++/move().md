- 它并不移动任何东西，只是把一个**左值强制转化为右值**，从而告诉编译器：“这个对象我以后不用了，你可以随便拆解它的内部资源献祭给别人。”

#### `std::move` 在类成员上的两个最经典场景
##### 1. 在构造函数的初始化列表中
当我们希望将传入的参数直接“存入”类成员，而不是先拷贝再赋值时
```cpp
class Player {
    std::string name;
public:
    // 将传入的临时字符串直接“移动”给成员变量
    Player(std::string n) : name(std::move(n)) {} 
};
```

##### 2. 在“设置”函数（Setter）中
如果你有一个函数负责更新成员变量，使用 `std::move` 可以避免不必要的内存分配，注意，此处不是`const T&`

```cpp
void setName(std::string newName) {
    name = std::move(newName); // 把新名字“夺取”过来，旧内存被自动处理
}
```
当我们写 `name = std::move(newName);` 时，我们实际上是在告诉编译器：“我已经不需要 `newName` 这个局部变量里的内容了，请直接把它的资源（比如堆内存地址）转移给 `name` 吧。”这样就省去了分配新内存和复制字符串字符的时间

- **不要在 `return` 时用 `std::move`**：编译器有“返回值优化（RVO）”，它比移动更快。你加了 `std::move` 反而可能干扰编译器的优化
- **不要**在函数内部写 `void set(const std::string& s) { id = std::move(s); }`

##### `const T&`和`std::move()`

|**特性**|**const T& (常量引用)**|**T + std::move (传值移动)**|
|---|---|---|
|**对待临时变量**|慢（被迫深拷贝）|**极快**（直接掠夺资源）|
|**对待普通变量**|**快**（一次深拷贝）|稍慢（拷贝 + 移动）|
|**代码整洁度**|传统，安全|现代，简洁|
|**适用对象**|所有对象|仅限支持移动的对象（如 `vector`, `string`）|
