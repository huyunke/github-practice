### 内存布局与引用的艺术
在 C++ 中，一切都建立在**实际内存**上

- **栈（Stack）：** 自动分配和释放。比如在函数里写 `int a = 10;`，它就在栈上。速度极快，但作用域结束就消失了
    
- **堆（Heap）：** 手动分配（`new`）。它一直存在，直到你手动 `delete` 它
    
- 当你传递一个大型的 `struct` 或 `class` 时，如果直接传值，C++ 会在内存中完整地**复制**一份。为了省钱（省内存和时间），我们通常使用**引用（&）**
```cpp
//⭐ 使用引用传递
void process(BigObject& obj);        // 可修改对象
void process(const BigObject& obj);  // 只读访问（最常用）
```

```cpp
//返回 const 引用（非常常见）
const std::string& getName() const {
    return name;
}
```
优点：

- 避免返回大对象拷贝
    
- 提供只读访问
    
- 常用于 getter

- 调用下面这个函数来处理包含100万个整数的`vector`时，程序必须在栈（或堆的相关部分）开辟一块同样大的空间，并把那一百万个数字逐个复制过去。这不仅浪费内存，还会消耗大量的 CPU 时间
```cpp
// 方式 A 
void processData(std::vector<int> data) { /* ... */ }
```
[悬停引用](悬停引用.md)

- 但对于所有的类型来说引用不一定是最好的，**小对象传值更好（比如int,char)**

- 原因如下：在底层，**引用（&）** 的实现通常是通过**指针**完成的

  - **传值**：直接把 `int` 的数值（通常 4 字节）放进寄存器或栈里
    
  - **传引用**：把该变量的**内存地址**（在 64 位系统上是 8 字节）传过去

  - 为了传递一个 `int`（4 字节）的引用，反而要传递一个 8 字节的地址，并且函数内部在使用它时还要通过地址进行一次“寻址”操作。这就像是为了送一张明信片，特意雇了一辆大卡车

1. **通常不应该对栈对象取地址使用，而是使用引用**，这是因为栈上的对象生命周期非常短，作用域一结束就消失了。在下面这个例子中
```cpp
int* getLocalPointer() {
    int x = 100; 
    return &x; 
}
//当 `getLocalPointer` 函数执行完，变量 `x` 所在的栈空间就被回收了，指针就指向了一块“荒地”（野指针）
```

### 左值与右值
1. 左值与右值的定义

- **左值（lvalue）**：有名字、有固定内存地址的对象。比如 `int a = 5;` 中的 `a`

- **右值（rvalue）**：临时的、没有名字的对象（**不能对其进行取地址操作**）。比如 `a + 5` 的结果，或者一个临时生成的字符串 `"hello"s`

2. `std::move()` 
   它并不移动任何东西，只是把一个**左值强制转化为右值**，从而告诉编译器：“这个对象我以后不用了，你可以随便拆解它的内部资源献祭给别人。”
   - **不要在 `return` 时用 `std::move`**：编译器有“返回值优化（RVO）”，它比移动更快。你加了 `std::move` 反而可能干扰编译器的优化。

### 继承、指针与内存布局
在 C++ 中，当一个子类（派生类）继承自父类（基类）时，子类的内存布局通常是：**父类的数据 + 子类特有的数据**

1. 对象切片（Object Slicing）

- 如果你尝试把子类对象直接赋值给父类对象：由于 `b` 的内存空间只有 `Base` 那么大，它装不下 `TBase` 多出来的部分。结果就是：`TBase` 特有的信息被“切掉”了

2. 多态与指针

- 为了保留子类的特性（实现多态），我们通常使用**指针**

- `Base* ptr = new TBase();` ✅ 这是安全的，因为指针只是一个地址

### 类型转换

| **转换类型**               | **绰号**  | **职责**                                                                                                           |
| ---------------------- | ------- | ---------------------------------------------------------------------------------------------------------------- |
| **`static_cast`**      | **理智派** | 用于**相关类型**之间的转换（如父子类指针转换、基本类型转换）。它在**编译时会检查这种转换是否合理**，适合子 → 父（上行转换），父 → 子（下行转换）虽然允许，但一般不推荐（除非你百分百确定父指针指向的是它的子对象） |
| **`reinterpret_cast`** | **硬来派** | 用于**完全无关类型**之间的位模式转换（如把 `int*` 转成 `A*`）。它不保证安全，只管重新解释内存里的二进制位                                                    |
| **`const_cast`**       | **特权派** | **唯一**能去掉（或加上）`const` 属性的转换。它只负责修改类型的“只读”标签                                                                      |
- **场景 1：去掉只读属性** 有一个 `const int*`，但接口要 `int*`
    
    - **工具**：`const_cast`
        
    - **注意**：这只是为了通过编译器检查。如果原始内存真的是常量，修改它会导致崩溃
        
- **场景 2：内存布局完全一样的 A 和 B** 你想把 `A*` 当成 `B*`
    
    - **工具**：`reinterpret_cast`
        
    - **本质**：这就像是给同一个地址换了个“说明书”
        
- **场景 3：当你想用统一接口处理不同对象时
    
    - **工具**：`static_cast`
        
    - **原理**：相比 C 语言的强制转换，它更安全，因为它会检查 `Base` 和 `TBase` 是否真的有继承关系，C风格的强制转化在背后其实偷偷尝试了 `static_cast`、`reinterpret_cast` 甚至 `const_cast`
    - **使用 `static_cast` 的好处**
        - **安全性检查**： 如果你不小心把一个指向 `Fruit` 的指针转成了 `Car*`，`static_cast` 会直接在**编译阶段报错**（因为它们没关系）。而 C 风格转换会默许这种错误，直到程序运行时崩溃。
    
        - **代码可读性（搜索性）**： 在庞大的项目中，如果你想找所有的危险转换，你只需搜索 `_cast` 关键字
    
        - **意图明确**： 你写出 `static_cast`，就是告诉后来维护代码的人：“我知道这两个类是有继承关系的，我是清醒的。”

### 无类型指针
在 C++ 中，`void*` 被称为**无类型指针**。它非常特殊：

- 它只记录一个**内存地址**，但它**不知道**那个地址上存的是什么（是 `int`？还是 `TBase`？它不关心）
    
- **用途**：当你编写一些非常底层的代码（比如内存拷贝函数 `memcpy`），或者需要与 C 语言库交互时，`void*` 是最好的中转站

- 但是不能直接对 `void*` 进行解引用。如果想用它，必须先用 `static_cast<int*>(vPtr)` 把类型“找回来”

### 取地址和引用
1. 语法上的区别

|**特性**|**引用 (T&)**|**取地址 (&T)**|
|---|---|---|
|**操作性质**|**声明**（定义一个新的变量名）|**运算**（执行一个动作获取结果）|
|**结果类型**|它就是对象本身的一个别名|它产生一个**指针** (`T*`)|
|**是否占空间**|逻辑上不占（它是现成对象的替身）|占空间（指针变量需要 8 字节存地址）|
|**是否可变**|**从一而终**（初始化后不能再引用别人）|**随遇而安**（指针可以指向 A 再改指向 B）|
