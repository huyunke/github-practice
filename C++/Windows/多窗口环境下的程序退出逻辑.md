在 `Windows `编程中，如果你有多个窗口，你通常希望只有在**主窗口**关闭时才调用 `PostQuitMessage`，而关闭其他子窗口（比如“设置”或“关于”对话框）时，只是隐藏或销毁该窗口，而不退出整个程序，有以下方法可以实现
#### 1.检查特定的窗口句柄 (HWND)
在你的 `WndProc` 中，你会收到当前正在处理消息的窗口句柄 `hWnd`。如果你在程序启动时记录了主窗口的句柄，就可以进行对比：
```cpp
if (hWnd == hMainWnd) // hMainWnd 是你在创建主窗口时保存的全局或静态变量
{
    PostQuitMessage(0);
}
```

#### 2.计算当前存活的窗口数量
你可以定义一个全局计数器（例如 `int g_windowCount`）。

- 每当创建一个新窗口时：`g_windowCount++;`
    
- 每当一个窗口收到 `WM_DESTROY` 时：`g_windowCount--;`
    
- **判断逻辑**：如果 `g_windowCount == 0`，说明这是最后一个窗口，此时调用 `PostQuitMessage(0)`
- 注意：采用这种办法是无法区分各窗口之间的区别
