默认窗口过程
- **作用**：它实现了 Windows 窗口的所有**标准行为**。如果你对某个消息不感兴趣，就把它丢给这个函数，它会帮你完成那些琐碎的系统级操作，但是它**不能做的是**“你程序的灵魂和业务逻辑”。
`DefWindowProc`包含的“标准行为”
- **非客户区 (Non-client Area) 绘制**：绘制窗口的标题栏、边框、右上角的（最小化、最大化、关闭）按钮。

- **窗口交互逻辑**：
    
    - 点击标题栏并拖动时，窗口随之移动。
        
    - 点击边框边缘时，鼠标变形状并允许拉伸窗口。
        
    - 双击标题栏实现最大化/还原。
        
- **默认鼠标指针管理**：当鼠标进入窗口范围时，根据注册窗口类时设置的默认光标（如箭头）进行切换。
    
- **系统菜单**：按下 `Alt + Space` 弹出的那个包含“移动”、“大小”、“关闭”的菜单。
    
- **键盘焦点管理**：处理按下 `Tab` 键在控件间切换的基本逻辑（虽然这通常需要更多协作）。
    
- **自绘触发**：当窗口被其他窗口遮挡后重新露出来时，它会向你的程序发送 `WM_PAINT`，如果你不处理，`DefWindowProc` 会用背景色刷一下屏幕（确保不留黑块），但它不知道该画什么具体内容。
