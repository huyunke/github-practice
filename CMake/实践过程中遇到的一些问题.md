### 在链接阶段找不到.lib
查了一下说是Windows动态库必须显式导出，需要在头文件中添加`__declspec(dllexport)`
```cpp
class __declspec(dllexport) Mathlib {  
public:  
    static int add(int a, int b);  
};
```
更加规范的做法：定义一个宏给头文件
```cpp
#ifdef MATHLIB_EXPORTS
#define MATHLIB_API __declspec(dllexport)
#else
#define MATHLIB_API __declspec(dllimport)
#endif

class MATHLIB_API Mathlib {
public:
    static int add(int a, int b);
};
```
`CMakelists.txt`添加
```cmake
target_compile_definitions(MathDynamic
    PRIVATE MATHLIB_EXPORTS
)
```
### 动态库链接成功之后程序运行后立即退出
动态库在链接成功后，程序运行时仍然需要能够找到对应的 `DLL` 文件。  
Windows 默认会在程序所在目录查找 DLL，因此通常需要将 `DLL` 复制到可执行文件的运行目录。  
可以在 `CMakeLists.txt` 中添加命令，在构建完成后自动复制 DLL🤗添加的命令如下所示
```cmake
add_executable(APP  
        main.cpp  
)  
  
add_dependencies(APP MathDynamic)  
target_link_libraries(APP PRIVATE MathDynamic)  

#需要添加的内容  
add_custom_command(TARGET APP POST_BUILD  
        COMMAND ${CMAKE_COMMAND} -E copy_if_different  
        $<TARGET_FILE:MathDynamic>  
        $<TARGET_FILE_DIR:APP>)
```
 `copy` vs `copy_if_different`

|命令|行为|适用场景|
|---|---|---|
|`${CMAKE_COMMAND} -E copy <src> <dst>`|每次构建都会无条件复制 DLL，即使目标文件和源文件完全一样|简单直接，但可能增加不必要的编译时间|
|`${CMAKE_COMMAND} -E copy_if_different <src> <dst>`|只有当源文件和目标文件内容不同时才会复制|更高效，避免重复拷贝|

💡 在大项目或者 DLL 很大的情况下，`copy_if_different` 可以节省构建时间。
