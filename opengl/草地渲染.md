## 1. 草的基本渲染方式（透明贴图 + 四边形）

### 核心思路

- 草通常不是高模，而是一个竖直四边形（Quad）
- 四边形上贴一张带透明通道的 PNG（RGBA）
- 在片元着色器中根据 alpha 丢弃透明像素

### 必要状态

- 开启深度测试：`glEnable(GL_DEPTH_TEST)`
- 开启混合：`glEnable(GL_BLEND)`
- 设置混合函数：`glBlendFunc(GL_SRC_ALPHA, GL_ONE_MINUS_SRC_ALPHA)`
- 草通常关闭剔除：`glDisable(GL_CULL_FACE)`（避免只看见单面）

## 2. UV 应该怎么设置

### 单张贴图映射

草片通常用标准 UV：

- 左下 `(0,0)`
- 右下 `(1,0)`
- 右上 `(1,1)`
- 左上 `(0,1)`

### 注意

- 草卡片不要用地板那种大于 1 的平铺 UV（如 0~10）。
- 如果纹理上下颠倒，检查 `stbi_set_flip_vertically_on_load(true/false)`。

## 3. Billboard（草始终朝向相机）

### 思路

每帧根据相机位置，为每株草构造“朝向相机”的旋转基底矩阵：

- `toCamera = normalize(cameraPos - grassPos)`
- `right = normalize(cross(worldUp, toCamera))`
- `up = cross(toCamera, right)`

然后把 `right/up/toCamera` 填到旋转基底矩阵中，再组合 `model`：

- `model = translate * billboardRotation * scale`

### 为什么 `right = normalize(cross(worldUp, toCamera))`

- `cross(a,b)` 结果垂直于 `a` 和 `b`
- 用世界上方向和朝向相机方向叉乘，可得到草局部坐标的“右轴”
- `normalize` 是为了去掉长度影响，只保留方向

## 代码
```cpp
glm::vec3 toCamera = glm::normalize(cameraPos - grassPos);  
toCamera.y = 0.0f;  
toCamera = glm::normalize(toCamera);  
  
glm::vec3 right = glm::normalize(glm::cross(glm::vec3(0.0f, 1.0f, 0.0f), toCamera));  
glm::vec3 up = glm::cross(toCamera, right);  
  
glm::mat4 billboardRot(1.0f);  
billboardRot[0] = glm::vec4(right, 0.0f);  
billboardRot[1] = glm::vec4(up, 0.0f);  
billboardRot[2] = glm::vec4(toCamera, 0.0f);  
  
glm::mat4 grassModel = glm::mat4(1.0f);  
grassModel = glm::translate(grassModel, grassPos) * billboardRot;  
grassModel = glm::scale(grassModel, glm::vec3(1.2f, 1.8f, 1.0f));  
grassShader.setMat4("model", grassModel);
```

## 4. 草抖动（风摆）实现思路

### 总体原则

- 在顶点着色器做位移，不在 CPU 改顶点。
- 根部少动，顶部多动。
- 不同草要有相位差，避免同步抖动。

### 常用参数（uniform）

- `uniform float uTime;`

- `uniform vec3 uWindDir;`

- `uniform float uWindStrength;`

- `uniform float uWindSpeed;`

### 关键公式

- 高度权重：`weight = clamp(aPos.y, 0.0, 1.0)`，再平方让顶部更明显。
- 相位差：`phase = worldPos.x * k1 + worldPos.z * k2`
- 摆动：`sway = sin(uTime * uWindSpeed + phase) * uWindStrength * weight`
- 位移：`worldPos.xz += normalize(uWindDir.xz) * sway`

## 5.其他：`clamp` 的含义

`clamp(x, minVal, maxVal)` 表示把 `x` 限制在 `[minVal, maxVal]` 区间：

- `x < minVal` 返回 `minVal`
- `x > maxVal` 返回 `maxVal`
- 否则返回 `x`

示例：`clamp(aPos.y, 0.0, 1.0)`  
表示把高度权重限制在 0 到 1 之间，防止摆动失控。
