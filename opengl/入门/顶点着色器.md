数据已经存进显存（VBO）了，但显卡现在就像拿到了一个没有任何说明书的包裹，它不知道该怎么处理这些数字。这时候我们就需要**顶点着色器 (Vertex Shader)** 🎨 来充当处理员

## 编写 GLSL 源代码
顶点着色器是用 **GLSL**编写的，以下是一个简单的着色器
```cpp
#version 330 core
layout (location = 0) in vec3 aPos; // 输入变量 aPos，位置在 0

void main() {
    gl_Position = vec4(aPos.x, aPos.y, aPos.z, 1.0); // 输出位置
}
```
- **`in` 关键字**：表示这是一个输入属性（从 VBO 传过来的）。
    
- **`vec3`**：代表一个包含 3 个 `float` 分量的向量（$x, y, z$）。
    
- **`gl_Position`**：这是 OpenGL 预定义的全局变量。着色器的最终目的就是给这个变量赋值，告诉显卡这个点在屏幕上的确切位置。
    
- **`vec4(..., 1.0)`**：OpenGL 内部使用的是**四维齐次坐标**。所以我们要把 3D 的 $x, y, z$ 后面补上一个 $1.0$。
    
> $w$ 分量（第四位）是用来处理 3D 透视效果的。

## 运行时编译
- **创建对象**：`glCreateShader(GL_VERTEX_SHADER)` 产生一个着色器 ID
```cpp
unsigned int vertexShader; 
vertexShader = glCreateShader(GL_VERTEX_SHADER);
```
    
- **关联源码**：`glShaderSource` 把我们的字符串代码传给这个 ID
```cpp
//第一个参数：着色器对象，第二个参数：传递源码字符串数量
//填 `NULL`：OpenGL 会认为你的字符串是以 `\0` 结尾的
glShaderSource(vertexShader, 1, &vertexShaderSource, NULL); 
```
    
- **开始编译**：`glCompileShader` 命令显卡开始处理这段代码
```cpp
glCompileShader(vertexShader);
```

## 错误检测
显卡编译代码时不会像编译器那样报错在控制台，如果代码写错了（比如少了个分号），画面可能直接变黑。所以我们需要手动检查编译是否成功。

- 使用 `glGetShaderiv` 获取 `GL_COMPILE_STATUS`。
    
- 如果失败，用 `glGetShaderInfoLog` 把报错的具体信息（Log）取出来打印
示例代码
```cpp
int  success;
char infoLog[512];
glGetShaderiv(vertexShader, GL_COMPILE_STATUS, &success);
if(!success)
{
    glGetShaderInfoLog(vertexShader, 512, NULL, infoLog);
    std::cout << "ERROR::SHADER::VERTEX::COMPILATION_FAILED\n" << infoLog << std::endl;
}
```
