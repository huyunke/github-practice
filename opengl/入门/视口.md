## 定义
OpenGL渲染窗口的尺寸大小

## 代码
调用`glViewport`函数来设置视口的**尺寸**
```cpp
glViewport(0, 0, 800, 600);
```
前两个参数控制窗口左下角的位置。第三个和第四个参数控制渲染窗口的宽度和高度

## 视口大小调整
```cpp
void framebuffer_size_callback(GLFWwindow* window, int width, int height) { 
	glViewport(0, 0, width, height); 
}
```
注册这个函数，告诉`GLFW`我们希望每当窗口调整大小的时候调用这个函数
```cpp
glfwSetFramebufferSizeCallback(window, framebuffer_size_callback);
```

## 视口的维度设置为比GLFW的维度小
### 1. 分屏游戏 

这是最直观的应用场景。如果你在做一个双人对战游戏，你需要将屏幕一分为二：

- **左侧玩家：** `glViewport(0, 0, width/2, height);`
    
- **右侧玩家：** `glViewport(width/2, 0, width/2, height);` 
- 通过两次渲染调用，配合不同的视口设置，可以在同一个窗口内绘制两个完全不同的摄像机视角

### 2. 编辑器 UI 布局 

像 Unity、Unreal Engine 或 Blender 这样的 3D 编辑器，窗口中只有一部分区域是真正的 3D 预览区，其余部分是工具栏、属性面板和层级树

- 你需要根据 UI 布局计算出 3D 视图的具体坐标和尺寸
    
- 只在指定的矩形区域内进行 OpenGL 渲染，防止渲染画面覆盖掉周围的 UI 控件

### 4. 保持固定纵横比

如果用户随意拉伸窗口，而你希望游戏画面始终保持 **16:9** 的比例，不被拉伸变形：

- 你需要计算出窗口内能容纳 16:9 画面最大的矩形区域。
    
- 如果窗口太宽，视口左右会留出黑边；如果窗口太高，视口上下会留出黑边。这种技术被称为 **Letterboxing**。

### 5. 小地图渲染 (Mini-map)

在 RTS 或 RPG 游戏中，角落的小地图通常是一个独立的渲染过程：

- 先渲染主游戏场景
    
- 修改视口到一个角落的小方块区域，渲染顶层俯视视角的地图数据
