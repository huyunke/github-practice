## 作用
是在显卡的内存（显存）中储存大量的顶点数据。这样做的好处是我们可以一次性把一批数据发送到显卡，而不是每个顶点发送一次，这能显著提高渲染效率

## 步骤
### 生成与绑定
在 OpenGL 中，我们首先需要一个 ID 来引用这个缓冲对象

- **生成**：使用 `glGenBuffers` 产生一个唯一的 ID，`glGenBuffers(1, &VBO)`。这里的 `1` 代表我们要创建一个缓冲，`&VBO` 是用来接收生成的 ID 的变量
    
- **绑定**：使用 `glBindBuffer` 将这个 ID 绑定到特定的“目标”上。对于顶点数据，这个目标通常是 `GL_ARRAY_BUFFER`。代码：`glBindBuffer(GL_ARRAY_BUFFER, VBO)`
```cpp
//一次性生成两个缓冲
unsigned int VBOs[2];
glGenBuffers(2, VBOs);
```
### 传输数据
 一旦绑定成功，我们就可以使用 `glBufferData` 函数把内存里的顶点数组（比如三角形的坐标）复制到显卡的缓冲内存中
 
 ---
 
它的参数结构如下： `glBufferData(target, size, data, usage)`

- **target**: 目标插槽，也就是我们刚刚绑定的 `GL_ARRAY_BUFFER`
    
- **size**: 传输数据的总大小。在 C++ 中，我们通常用 `sizeof(vertices)` 来自动计算
    
- **data**: 实际数据的指针（比如你的顶点数组名）
    
- **usage**: 告诉显卡你怎么使用这些数据（详见管理内容）


### 管理模式
显存的读写速度非常快，但空间有限。我们需要通过第四个参数告诉显卡我们打算如何使用这些数据，以便它进行优化：

- `GL_STATIC_DRAW`：数据几乎不变，例如：游戏地图里的山脉、建筑。显卡会把它放在读取速度极快但修改麻烦的地方
    
- `GL_DYNAMIC_DRAW`：数据会频繁改变，例如：受物理引擎影响不断变形的布料。显卡会把它放在方便 CPU 频繁写入的地方
    
- `GL_STREAM_DRAW`：数据每次绘制都会改变，例如：实时变化的粒子特效
