## 1. 调试工具栏主面板

第一张截图中显示的横向图标栏，从左到右依次为：

| **名称**       | **功能描述**                                  |
| ------------ | ----------------------------------------- |
| **重新运行 **    | 停止当前调试并以相同配置重新启动。常用于修改代码后快速重新测试           |
| **停止 **      | 彻底结束当前的调试会话                               |
| **恢复程序 **    | 让程序继续运行，直到遇到**下一个断点**。如果后面没有断点，程序将直接运行到结束 |
| **暂停程序 **    | 强制暂停正在运行的程序（即使没碰到断点），查看当前执行到了哪一行          |
| **步过 **      | 执行当前行并跳转到下一行。**不会进入**当前行调用的函数内部           |
| **步入 **      | 进入当前行所调用的函数内部，从函数的第一行开始调试                 |
| **单步执行我的代码** | 指程序一次只运行一行代码的**调试状态**                     |
| **步出 **      | 完成当前函数的剩余部分，跳回到调用该函数的地方                   |
| **查看断点**     | 打开断点管理窗口，可以启用/禁用或设置条件断点（如：`i > 10` 时才停下）  |
| **忽略断点**     | 暂时忽略所有断点，让程序无阻碍运行                         |

## 2. 高级操作菜单

这是第二张截图中出现的下拉菜单，包含了一些更高级的控制技巧：

- **强制步过 (Force Step Over):** 即使下一行代码里有断点，也会直接跳过去，不触发断点
    
- **智能步入 (Smart Step Into):** 如果一行代码是 `funcA(funcB())`，点击这个可以让你选是进 `funcA` 还是 `funcB`
    
- **运行到光标处 (Run to Cursor):** 非常实用。不需要打断点，直接让程序运行并停在你鼠标点击的那一行
    
- **显示执行点 (Show Execution Point):** 如果你在代码里翻页找东西迷路了，点这个图标能瞬间**跳回**当前程序正卡在那里的那一行
    
- **对表达式求值 (Evaluate Expression):** 调试的神器。你可以输入任何代码（比如 `my_list.length()` 或 `user.name == 'admin'`）来实时计算结果，而无需修改原代码
